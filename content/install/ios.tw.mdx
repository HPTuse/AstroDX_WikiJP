<Callout title="接下來是客製化歌曲系列的教學">
  若您沒有想要客製化歌曲系列，可以直接**跳過此段文章**，或甚至直接關閉此維基（因為譜面已安裝完成）
</Callout>
在 AstroDX 資料夾內有兩個需要注意的資料夾：`collections` 和 `levels`。
在 `collecitons` 資料夾中僅有儲存歌曲系列的資料，而在 `levels` 中主要儲存譜面本身。

若欲更改歌曲系列，請編輯 [`collections/manifest.json`](https://astrodx.notion.site/Collection-Manifest-Details-8f966de6971748e0ac01c0a0a6054594) 檔案。

但簡單而言，在 `manifest.json` 中會有這種格式：
```json
{
  "name": "歌曲系列名稱",
  "id": null, // 請忽略
  "serverUrl": null, // 請忽略
  "levelIds": ["在此的每個曲目", "都是在 levels 內的", "資料夾名稱"]
}
```

    如果您現正使用 Beta 2 版號 4 或更新的版本，我們推薦使用[這種安裝方式](#installing-levels)。
    </Accordion>

    <Accordion title="有沒有辦法在打音遊時暫時關閉通知中心？常常會不小心滑到。">

    </Accordion>

    <Accordion
    title="Information regarding file compatibility"
    >
    I couldn't phrase the question right, so I'm just gonna dump this info here.

    The officially supported types of codecs for AstroDX on iOS:
    	- **Video:** H.264
    	- **Audio:** OGG Vorbis (with .ogg extension) **OR**  mp3 (with .mp3 extension)
    </Accordion>

<Accordion title="我的 PV 已經放在資料夾內了，但它沒有在播放！">
	請檢查是否在設定中關閉 PV 設定，並且確認 PV 的檔案名稱為 `pv.mp4`。順代一提，`mv.mp4` 是專門在
    [Majdata](https://rentry.org/maiguide) 跑的，無法在 AstroDX 中通用。
    </Accordion>

<Accordion title="為什麼 AstroDX 還沒在 App Store 上？">
	我們希望將程式碼改到一定的程度在上架，況且目前還是在雛型的階段。
	加上在 App Store 上上架要錢，所以目前沒有上架的打算。
    </Accordion>

<Accordion title="為什麼你們不編譯 `.ipa` 檔？">

我們不這麼做是基於幾個原因：

    > 目前我們不建議側載 `.ipa` 檔，加上 `.ipa` 檔十分容易被竄改，然後被有意人士放在 App Store
    > 上。為了防止其發生我們暫時不提供 `.ipa` 檔。

    </Accordion>

    <Accordion title="為什麼 Testflight 的測試欄位那麼容易滿？">
    太多人想玩 AstroDX 了。
    </Accordion>
    <Accordion title="為什麼不建立更多測試組別？">
    建立這些組別要錢。
    </Accordion>


<Accordion title="[2.0 Beta] 為什麼資料夾結構跟以往不同？">
在前幾個版本的 AstroDX 中，歌曲可以被直接放在 `levels` 資料夾，或是放在 levels
    內的子資料夾當作一個**分類**。不過這種方便的管理方式也產生了不少問題。

    **多個分類中不可有相同的歌曲名稱**
    因為分類都是以子資料夾的形式存在，將相同的歌曲一次放置在多個不同分類是有難度的，更何況是移除、管理、變更一個歌曲的內容。

	**不同歌曲可以有相同的名稱**
    即使歌曲是用不同的子資料夾區分類別，但在遊戲中仍會合併為一份清單。這使得不同分類內的相同歌曲產生衝突。

	這些問題使得我們必須使用新的分類方式作為替代方案。
    雖然此分類方式較為複雜，但將來遊戲內將有分類編輯器，簡化管理分類的流程。
</Accordion>
